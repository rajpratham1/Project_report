Project Report: Breast Cancer Detection

Abstract

This project presents an advanced machine learning system for automated breast cancer detection using the Wisconsin Breast Cancer Dataset. Leveraging a Random Forest classifier, the system achieves high diagnostic accuracy. Key features include data preprocessing with SMOTE for class imbalance and a user-friendly web interface built with Flask, designed to assist healthcare professionals in making faster, more informed decisions. The system demonstrates the effectiveness of machine learning in enhancing medical diagnostics.

1. Introduction

Breast cancer is one of the most common and life-threatening cancers affecting women globally. Early and accurate detection is paramount to improving survival rates and treatment outcomes. Traditional diagnostic methods, while effective, can be time-consuming and subject to human variability.

This project aims to address this challenge by developing an intelligent, machine-learning-based system to assist healthcare professionals. By providing an AI-powered diagnostic tool, we seek to enhance the accuracy and efficiency of breast cancer detection, offering a reliable second opinion and streamlining the diagnostic workflow.

2. Objectives

The primary goals for this project were clearly defined:

  • Develop a highly accurate machine learning model for the binary classification of breast tumors (benign vs. malignant).
  • Create an intuitive web interface using Flask that allows users to get predictions for single or batch data entries.
  • Implement robust data preprocessing, including feature scaling and handling class imbalance using SMOTE.
  • Implement batch processing capabilities to allow for the prediction of multiple samples from an uploaded file.
  • Ensure the system is well-structured and can serve as a practical tool for educational or assistive diagnostic purposes.

3. Dataset Description

The model was trained and tested using the well-known Wisconsin Breast Cancer Dataset, a reliable benchmark for classification tasks.

  • Source: UCI Machine Learning Repository
  • Total Samples: 569 cases
  • Features: 30 numerical measurements (e.g., radius, texture, smoothness) computed from a digitized image of a fine needle aspirate (FNA) of a breast mass.
  • Classes: 2 (Binary)
    • Malignant (Cancerous)
    • Benign (Not Cancerous)

4. Methodology

The process of building the model involved several key stages, from preparing the data to selecting and training the final classifier.

4.1. Data Preprocessing

Raw data is rarely ready for training. Our preprocessing pipeline included two critical steps:

  • Feature Scaling: We applied StandardScaler from scikit-learn to normalize the 30 numerical features. This ensures all features contribute equally to the model's decisions, preventing features with larger scales from dominating.
  • Handling Class Imbalance: The dataset is imbalanced. To prevent the model from being biased towards the majority class, we used the SMOTE (Synthetic Minority Over-sampling Technique) to generate synthetic samples for the minority (malignant) class, creating a balanced training set.

4.2. Model Selection

After experimenting with several classifiers, the RandomForestClassifier was chosen for its high performance, robustness against overfitting, and ability to handle complex, non-linear relationships. The model was configured with 100 trees (n_estimators=100) and the Gini impurity criterion.

4.3. Training Process

The dataset was split into training (80%) and testing (20%) sets. We employed GridSearchCV for hyperparameter tuning to find the optimal settings for the Random Forest model. The model was then trained on the preprocessed and oversampled training data and evaluated on the unseen test set.

5. System Architecture

The web application is built on a simple and effective three-tier architecture to separate concerns and ensure maintainability.

  • Presentation Layer (Front-End): This is the user interface built with HTML5, CSS3 (style.css), and JavaScript (script.js). It provides the interactive forms for single and batch predictions, theme toggling, and displays the results from the backend.
  • Application Layer (Back-End): A Flask-based server (app.py) acts as the system's brain. It handles incoming web requests, processes data, and exposes REST API endpoints:
    • /predict: For single predictions from the web form.
    • /predict_batch: For handling batch file uploads.
  • Data Layer (Model): This layer consists of the pre-trained machine learning model (arya_best_cancer.joblib), which is a saved instance of our trained RandomForestClassifier. Data processing libraries like Pandas and NumPy are used here to format incoming data before feeding it to the model.

6. Results & Evaluation

The final model was evaluated on the unseen 20% test split. The system demonstrated excellent performance, validating the chosen methodology and model.

The key performance metrics are summarized below:

Metric Score
Accuracy 97.4%
Precision (Malignant) 98.6%
Recall (Malignant) 96.5%
F1-Score (Malignant) 97.5%
ROC AUC Score 99.64%

The high scores, particularly the 97.4% accuracy and the near-perfect 99.64% ROC AUC, indicate that the model is highly effective at distinguishing between benign and malignant tumors with a very low error rate.

7. Visualizations

During the exploratory data analysis (EDA) phase in the cancer_model_notebook.ipynb, several key visualizations were created to understand the dataset:

  • Correlation Heatmap: A heatmap was generated to show the correlation between all 30 features. This helped identify multicollinearity and understand which features were strongly related to each other and to the target 'diagnosis'.
  • Pairplot: A pairplot from the Seaborn library was used to visualize the relationships between a subset of key features (e.g., 'radius_mean', 'texture_mean', 'perimeter_mean'). This plot helped to visually confirm the separation between the 'benign' and 'malignant' classes based on these features.

These visualizations were crucial for confirming the dataset's suitability for machine learning and guiding the feature selection and preprocessing steps.

8. Conclusion

This project successfully demonstrates the application of machine learning in medical diagnostics, culminating in a highly accurate (97.4%) breast cancer detection system. By integrating a tuned Random Forest model with a user-friendly Flask web application, we have created a practical tool that can effectively assist healthcare professionals.

The system's ability to handle both single and batch predictions, combined with a robust preprocessing pipeline using SMOTE, showcases a comprehensive approach to solving a real-world problem. This project serves as a strong proof-of-concept for the value of AI-powered diagnostic tools in improving patient outcomes through early and accurate detection.

9. Future Scope

While the current system is highly effective, there are several avenues for future enhancement:

  • Integration with Medical Imaging: Extend the system to accept medical images (e.g., mammograms) and use Deep Learning (CNNs) for direct image-based classification.
  • Multi-class Classification: Adapt the model to classify different sub-types of breast cancer, not just the binary benign/malignant.
  • Real-time API Deployment: Deploy the model on a cloud platform (like AWS or GCP) for a scalable, real-time API that can be integrated into existing hospital information systems.
  • Explainable AI (XAI): Implement XAI techniques (like LIME or SHAP) to provide explanations for each prediction, increasing trust and interpretability for medical staff.
